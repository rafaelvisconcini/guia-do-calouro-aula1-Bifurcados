# Evidências do trabalho

A nota leva em conta o **processo**: o que você fez no Git e no GitHub. Para comprovar isso, cada
aluno mantém um registro de evidências enquanto trabalha.

## O que registrar

Durante cada aula, vá juntando:

- **Prints (capturas de tela)** dos passos importantes: o terminal depois de um commit, a sua branch
  no GitHub, o Pull Request aberto, a revisão aprovada, o merge, o conflito e a resolução.
- **Os comandos** que você rodou (copie do terminal).
- **Anotações da aula**: o que o professor explicou, dúvidas que apareceram e como foram resolvidas.

## Como organizar

1. Copie o modelo para um arquivo com o seu @usuario:

   ```bash
   cp evidencias/MODELO.md evidencias/SEU-USUARIO.md
   ```

2. Preencha esse arquivo ao longo da aula. Para imagens, crie a pasta `evidencias/imagens/` e
   referencie assim: `![meu commit](imagens/meu-commit.png)`.
3. Commite junto com o resto do trabalho — é mais um exercício de Git.

## Cuidado

Não inclua prints com **senhas, tokens ou dados privados** de ninguém. O repositório é público.
